Property: filterJSON (default is null)
This property can take a boolean value as well as an object notation. Setting this property to anything other than null forces the Paginated Drop Down Widget to use the jQuery filterJSON plugin. As the name suggests, filterJSON plugin can be used to filter a retrieved JSON Object.
#Setting this property to true will use filterJSON plugin with default values such as below: 
{
	property: same as valueProperty,
	value: same as text box value,
	checkContains: true,
	wrapper: true
}
#Setting this property to a valid filterJSON config would force the Paginated Drop Down Widget to extend the default filterJSON config (as shown above).
**An important thing to mention over here is the 'wrapper' property of filterJSON plugin. If it is set to true (as the case should always be) it will return the matched properties containing object. If it is set to false, then the property itself is returned. However, you are free to pass any valid filterJSON config to the widget as per your requirement.
Demo: default behaviour
Sample Code:
$( "#x-demo-1" ).paginateddropdown({
	url: "assets/cricketers.json",
	valueProperty: "name",
	itemTemplate: "<div title='{@:email}'><div>{@:name}</div></div>"
});
Demo: Setting filterJSON to true
Sample Code:
$( "#x-demo-2" ).paginateddropdown({
	url: "assets/cricketers.json",
	valueProperty: "name",
	itemTemplate: "<div title='{@:email}'><div>{@:name}</div></div>",
	filterJSON: true
});
Demo: Setting filterJSON to config object
Sample Code:
$( "#x-demo-3" ).paginateddropdown({
	url: "assets/cricketers.json",
	valueProperty: "name",
	itemTemplate: "<div title='{@:email}'><div>{@:name}</div></div>",
	filterJSON: {startsWith: true}
});
